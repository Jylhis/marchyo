## Creating a new module

### 1. Create the module file

Create a new file in the appropriate directory:

- `modules/nixos/` — System-level configuration
- `modules/home/` — User-level configuration
- `modules/generic/` — Shared between both

### 2. Write the module

Follow the standard module structure:

```nix
{ config, lib, pkgs, ... }:
let
  cfg = config.marchyo;
in
{
  config = lib.mkIf cfg.myFeature.enable {
    # Your configuration here
    environment.systemPackages = with pkgs; [
      some-package
    ];
  };
}
```

For `modules/nixos/` and `modules/home/`, that's it — both directories are auto-discovered (via `lib/discover-modules.nix`), so there is no import list to edit. Module merging is order-independent; reach for `lib.mkBefore`/`lib.mkAfter` or priorities only if a specific merge order matters. The one exception is `modules/darwin/`, whose `default.nix` is a hand-curated import list you must edit manually.

### 3. Define options

Add any new options under `modules/nixos/options/` — a directory of files split by namespace, each declaring `options.marchyo.<namespace>`. The directory is auto-discovered, so add to an existing namespace file or create a new one (e.g. `myfeature.nix`):

```nix
{ lib, ... }:
{
  options.marchyo.myFeature = {
    enable = lib.mkOption {
      type = lib.types.bool;
      default = false;
      description = "Enable my feature";
    };
  };
}
```

### 4. Add a test

Drop or extend a file under `tests/eval/`:

```nix
{ helpers, ... }:
let
  inherit (helpers) testNixOS withTestUser;
in
{
  eval-my-feature = testNixOS "my-feature" (withTestUser {
    marchyo.myFeature.enable = true;
  });
}
```

## Home Manager modules

Home Manager modules that need NixOS config use `osConfig`:

```nix
{ config, lib, osConfig ? {}, ... }:
let
  cfg = osConfig.marchyo or {};
in
{
  config = lib.mkIf (cfg ? myFeature && cfg.myFeature.enable) {
    # User-level configuration
  };
}
```

## Best practices

- **Use `lib.mkIf`** for feature-gated configuration blocks
- **Use `lib.mkDefault`** for values consumers should be able to override
- **Use `lib.mkMerge`** when a module has multiple conditional branches
- **Define all options under `modules/nixos/options/`** — never define `marchyo.*` options in individual modules
- **Add a test** for every new feature flag
