title: Theming
description: Customize system appearance with Stylix and base16 color schemes

Marchyo uses [Stylix](https://github.com/danth/stylix) for consistent theming across GTK, Qt, terminal applications, and the Hyprland compositor.

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `marchyo.theme.enable` | `bool` | `true` | Enable the Stylix theming system |
| `marchyo.theme.variant` | `"dark"` or `"light"` | `"dark"` | Theme variant preference |
| `marchyo.theme.scheme` | `string`, `attrs`, or `null` | `null` | Override the base16 color scheme |

## Basic usage

```nix
marchyo.theme = {
  enable = true;
  variant = "dark";
};
```

## Color schemes

By default Marchyo themes the system with the Jylhis design palette, computed from the upstream design tokens and selected by `marchyo.theme.variant` (a dark or light rendering of the same palette).

To use a different base16 scheme, set `marchyo.theme.scheme` to the name of a scheme shipped by [`base16-schemes`](https://github.com/tinted-theming/base16-schemes):

```nix
marchyo.theme = {
  variant = "dark";
  scheme = "nord";
};
```

When `scheme` is set, Marchyo loads `${pkgs.base16-schemes}/share/themes/<scheme>.yaml` instead of the Jylhis palette.

## Light mode

To use the light theme:

```nix
marchyo.theme = {
  variant = "light";
};
```
