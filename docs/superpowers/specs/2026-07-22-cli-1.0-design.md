# Marchyo CLI 1.0 — Design

Date: 2026-07-22
Status: approved (brainstorming session)
Baseline: marchyo-cli 0.1.0 (post PR #121 — system subcommands shipped)

## 1. Definition of 1.0

marchyo CLI 1.0 is the release where `marchyo` becomes the single desktop
control surface for a marchyo system. It ships when all six criteria hold:

1. **Runtime-first change model (F3.0):** every mutating desktop command
   applies live by default, persists declaratively with `--apply`
   (write `cli-state.json` + `nixos-rebuild`), and undoes with `--revert`.
2. **Full command surface (F3.2):** the omarchy-parity groups exist —
   system, theme, toggle, capture, menu, power, media/font, declarative.
3. **Multi-theme runtime switching (F3.3):** `theme list|set|next` operates
   over a user-declared theme list (`marchyo.theme.themes`), default the two
   Jylhis variants.
4. **Script absorption complete:** all `marchyo-*` helper-script logic lives
   in TypeScript; Hyprland binds, waybar handlers, and menu actions invoke
   `marchyo …` directly; the `writeShellApplication` packages are deleted.
5. **Compatibility contract frozen:** command names, arguments, flags, and
   exit codes (`0/1/2`) are stable until 2.0. State files
   (`/etc/marchyo/cli-state.json`, `~/.local/state/marchyo/runtime.json`)
   remain **internal**, schema-versioned, and may change between minors.
6. **Release blockers cleared:** CI runs the CLI test suite; marchyo.org
   documents every command; bash/zsh/fish completions and a man page install
   with the Nix package.

Version jumps 0.1.0 → 1.0.0, single-sourced from `package.json`.
Explicit non-goals for 1.0: the dev-CLI split (`marchyoctl` stays installed
system-wide), local AI (F1.11), darwin support for the user CLI.

## 2. Sequencing: engine-first vertical slices (approach A)

Build the F3.0 engine once, prove it on the theme slice, then port one
command group per PR. Every PR ports logic, flips the binds it owns, deletes
its scripts, and adds tests — the system is fully working after each merge.
Contract freeze + docs/CI/completions land last, then tag.

## 3. F3.0 engine (packages/core)

Three new modules, consumed by every mutating command:

- **`runtime-state.ts`** — owns `~/.local/state/marchyo/runtime.json`:
  `{ "schemaVersion": 1, "overrides": { "<dotted.key>": <value> } }`.
  Atomic writes (temp file + rename). Overrides are ephemeral by design:
  cleared on `nixos-rebuild` activation (the contract `theme-runtime.nix`
  established in #118).
- **`desktop.ts`** — typed actuator wrappers: `hyprctl` (keyword get/set,
  reload, dispatch), `makoctl`, `systemctl --user`, `swayosd-client`. All
  exec through the existing `system.ts` runner so tests inject mocks.
- **`apply.ts`** — the three-mode orchestrator. Commands register a
  `ChangeSpec { key, runtimeApply(), runtimeRevert(), stateWrite() }`;
  `apply.ts` handles mode selection and runtime.json bookkeeping, and for
  `--apply`/`--revert` delegates to `state.ts` (cli-state.json) and `nix.ts`
  (rebuild). One implementation, N commands.

New plumbing command: **`marchyo runtime status|restore`** — `status` lists
active overrides; `restore` re-applies them. `restore` is wired as a
Hyprland `exec-once` + post-reload hook so overrides survive
`hyprctl reload`, replacing per-script hacks.

## 4. Command surface at 1.0 (frozen)

| Group | Commands | Mode |
|---|---|---|
| System (exists) | `status`, `rebuild`, `update`, `upgrade`, `rollback`, `gc`, `diff`, `debug` | as today |
| Theme | `theme list`, `theme get`, `theme set <name>`, `theme next`, `bg set <path>`, `bg next` | runtime-first |
| Toggle | `toggle <name> [on\|off]`: `gaps`, `transparency`, `nightlight`, `waybar`, `touchpad`, `touchscreen`, `idle`, `screensaver`, `notifications`, `suspend`, `hybrid-gpu` | runtime-first; `hybrid-gpu` is `--apply`-only |
| Capture | `capture screenshot [--edit]`, `capture record [--audio none\|desktop\|mic]`, `capture ocr`, `capture color` | action (stateless) |
| Menu | `menu`, `keybindings`, `launch <app>`, `focus-or-launch <app>` | action |
| Power | `lock`, `logout`, `suspend`, `reboot`, `shutdown`, `powerprofile get\|list\|set <p>` | action |
| Media/font | `transcode <file> [--ascii]`, `font list\|current\|set <name>` | `transcode` action; `font set` runtime-first |
| Declarative | `install <feature>`, `remove <feature>`, `webapp add <url> [--name --key]`, `webapp rm <name>`, `security enroll fido2\|fingerprint` | `--apply`-only |
| Runtime | `runtime status`, `runtime restore` | plumbing |

Contract rules:

- Every mutating command accepts `--apply` / `--revert`, or is documented
  `--apply`-only (declarative group, `hybrid-gpu`).
- Global flags (`--format text|json`, `--json`, `--color`, `--no-color`,
  `--plain`, `--no-animation`, `--no-input`, `-q`, `-v`) and exit codes
  `0` success / `1` runtime failure / `2` usage error stay exactly as today.
- Back-compat: `theme set dark|light` keeps working — `dark`/`light` are
  aliases for the Jylhis variants. `--rebuild` becomes a deprecated alias
  for `--apply` (warns on use; removed in 2.0).

## 5. Script absorption map (five port slices)

| Slice | Scripts absorbed | HM modules touched |
|---|---|---|
| 1. Theme (engine proof) | `marchyo-theme-toggle` | `theme-runtime.nix` |
| 2. Toggles | `marchyo-dnd-toggle`, window-toggle scripts, idle/screensaver toggles | `window-toggles.nix`, `screensaver.nix`, `waybar.nix` |
| 3. Capture | screenshot/OCR/record/hyprpicker wrappers | `screenshot.nix` |
| 4. Menus/power | `marchyo-menu`, `marchyo-power-menu`, `marchyo-keybindings` | `menus.nix`, `keybindings-cheatsheet.nix` |
| 5. Utilities | `marchyo-transcode`, `marchyo-share`, reminders, quick-info | `utilities.nix` |

Rules:

- Each port PR flips the binds it owns to `marchyo …` and deletes the
  absorbed script packages in the same PR (no wrapper shims, no alias debt).
- `modules/home/*` keep owning binds and window rules; they reference
  `pkgs.marchyo-cli` instead of bespoke `writeShellApplication`s.
- The gum-based floating-ghostty menu *presentation* stays as-is; the
  actions those menus dispatch become `marchyo …` calls.
- Out of scope: waybar dictation module, AI tooling scripts (not
  omarchy-parity surface), anything that must run before the CLI is on PATH
  (none known today).

## 6. Multi-theme system (F3.3)

- New option **`marchyo.theme.themes`** (list of names), default
  `[ "jylhis-dark" "jylhis-light" ]`. Jylhis entries derive from the design
  tokens as today; any other name resolves through the existing
  `marchyo.theme.scheme` mechanism (`base16-schemes` YAML).
- Build generalizes the #118 per-variant layout from 2 to N: each listed
  theme gets a store directory of pre-generated assets (waybar CSS, ghostty,
  mako, GTK, fzf, bat, starship, hyprlock colors) plus a build-generated
  manifest JSON that `theme list` reads.
- `theme set <name>` at runtime = swap the current-theme pointer symlink +
  reload consumers (proven #118 mechanism). `theme next` cycles manifest
  order. `--apply` writes the selection into `cli-state.json` + rebuild.
- Documented limitation (carried from #118): runtime switching covers the
  directly-themed surfaces; Stylix-derived targets (qt, …) follow only
  after an `--apply` rebuild.
- `bg set|next` cycles wallpapers via the existing wallpaper infrastructure,
  per-theme where a theme provides a set, falling back to
  `marchyo-wallpapers`.

## 7. Testing

- **Unit (bun):** every command via the existing mocked-exec pattern. The
  engine gets an exhaustive mode-matrix test (runtime/apply/revert ×
  override present/absent).
- **Contract tests:** snapshot tests over `--help` trees and
  `--format json` outputs of frozen commands; CI fails on accidental
  surface changes — this enforces criterion 5.
- **Eval (nix):** extend `tests/eval/cli.nix` and `tests/eval/theme.nix`
  for `theme.themes` and each new option. Options stay declaration-only
  and platform-neutral (darwin eval gate).
- **VM:** per-slice manual checklist via `just run` — each port PR verifies
  its binds live. Never auto-`nixos-rebuild` in CI; system commands are
  covered via `--dry-run`.

## 8. Release infrastructure

- **CI:** new path-filtered workflow on `packages/marchyo-cli/**`:
  `bun install --frozen-lockfile` → `bun run typecheck` → `bun test`
  (mirrors `site.yml`).
- **Completions + man:** a `marchyo completion <shell>` subcommand
  generates bash/zsh/fish; `package.nix` runs it at build and installs the
  outputs plus a man page generated from the command tree — zero drift by
  construction.
- **Docs:** new `site/src/content/docs/docs/usage/cli.mdx` (full command
  reference, `marchyoCliState` opt-in snippet, the `--impure` story);
  `hotkeys.mdx` regenerated once binds call `marchyo`; `updating.mdx` gains
  `marchyo update/upgrade/rollback`.
- **Maintenance:** `just cli-repin` recipe automating the FOD
  `outputHash` two-step after dependency changes.
- **Versioning/release:** 1.0.0 single-sourced from `package.json`
  (`version.ts` and `package.nix` read it); tag `marchyo-cli-v1.0.0` when
  the six criteria are green.

## 9. Decisions log (from the brainstorming session)

| Decision | Choice |
|---|---|
| 1.0 scope | All of F3.0 + F3.2 + F3.3 |
| Stability contract | Commands/flags/exit codes frozen; state files internal |
| Release blockers | CLI CI, website docs, completions + man page (dev-CLI split deferred) |
| Script strategy | Absorb logic into TypeScript |
| Migration | Binds call `marchyo` directly; scripts deleted per port PR, no shims |
| Theme source | User-declared `marchyo.theme.themes` list, Jylhis pair default |
| Sequencing | A: engine-first vertical slices |

## Addendum (frozen surface, PR 10)

Surface additions discovered while absorbing the real script inventory,
folded into the 1.0 freeze: `share`, `reminder set|show|clear`,
`info datetime|battery`, `monitor scale-cycle|laptop-toggle`, `zoom`,
`hibernate`, `focus-or-launch`, capture `--target`/`--edit`,
`transcode --to`, `font list|current|set`, `menu [power]`,
`completion bash|zsh|fish|man`. The `bg next` wallpaper-set fallback was
not needed (it cycles the wallpaper package directory). The dev-CLI split
remains deferred (out of the six release criteria).
